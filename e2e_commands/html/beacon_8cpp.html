The exact method we're going to use to send a beacon is still sort of up in the air. There are a few candidates:

  1. Switch the AX5043 into ASK mode and send out characters as data. The issue is that the AX5043 framer is putting a small amount of trash in the output. The advantage is that it 'works' with my little test boards because it can be implemented (the test boards don't have any RF switches or a PA)
  2. Set the AX5043 into CW mode and use the RF switches to vary the power level, leaving the PA on (will probably about 20dB on/off ratio. Really easy to do.
  3. Do the same as 2, but switch the PA on/off too. Need to verify that PA is okay with a signal at the input, but as long as we sequence the switch (before turning on the PA) it should be okay.

this is also a placeholder until we get code to convert ASCII to Morse code. Don't know what that's outputting. Been told it exists, but no idea where it is. assuming that we will set up radio and transmit it here and then return it back to the RX state when complete for now this code is based on method 1, although I think method 3 will likely be used in the long run...it's easier and won't have any glitches in the output.

for method 3, the radio needs to be in wire mode. You then set the modulation to FSK (already is) and the deviation to 0 Hz, and the datrate to ALMOST 0 (0,0,1)(on the TXRATE registers). Then to jump to wire mode write 0x84 to PINFUNCDATA (there's a library call for that) and set to Full_TX. Still need to try it though

okay we want to just send some Morse characters what so we would normally use a big case statement, but I don't feel like implementing that right now, so let's just do enough to show it works use the letters, not the prosign, so there's a 1 interval space after each character
