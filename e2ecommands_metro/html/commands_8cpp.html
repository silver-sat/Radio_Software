what's in the command buffer is KISS encoded, but guaranteed by design to be ASCII printable characters, so there's no need to decode it. a command packet has multiple parts (but we don't really care about a lot of it) KISS delimiter, KISS command byte, maybe some data, KISS delimiter length in most cases will be 1 (should test that too)
